## Successes and Strengths

**Strong vertical-slice delivery**
Every feature shipped as a connected workflow — auth, onboarding, class setup, invite/join, and COPPA consent all landed as end-to-end user journeys rather than isolated backend tasks. The teacher can sign in, complete onboarding, create a class, add students, and generate invite links in a single session.

**Multi-role UX coherence**
Teacher, parent, student, and admin onboarding paths stayed role-specific while maintaining consistent visual and interaction patterns. Getting this right in a foundation epic prevents expensive re-alignment later.

**Security and compliance foundation landed early**
Tenant scoping, role enforcement (RBAC), COPPA consent mechanics, and invite link security were all built before feature load increases in Epic 5+. The 3 High security findings caught in Story 4.8 code review (cross-tenant invite link, silent deep-link error swallowing, double-prefixed IDs) were all resolved before sign-off.

**Cross-story knowledge propagation**
Each story's Dev Notes carried a "Previous Story Intelligence" block that passed hard-won discoveries forward — the MinIO `wget` not `curl` gotcha (4.2), `expire_on_commit=False` SQLAlchemy async pattern (4.3), design token name corrections (4.5), Google Sign-In v16 SDK breaking changes (4.6). No discovery was made twice.

**Velocity curve improved across the epic**
Stories 4.1–4.3 had the most friction (tooling, new patterns). By Story 4.9 the completion note read "implementation proceeded without blockers." The accumulated context made later stories significantly smoother.

**elephant's proactive infrastructure debugging**
Before the retrospective, elephant independently diagnosed and fixed two critical blockers:
- Stale Alembic revision `0003` missing from codebase (DB state diverged from migrations)
- Container using host virtualenv causing `ModuleNotFoundError: starlette` at startup
- MinIO API returning unhealthy status

These fixes were non-trivial and prevented the retro from being blocked on infrastructure.

**Validation culture held**
Repeated typecheck and API test runs (304 tests, all green) caught schema drift quickly and prevented quality regression across 10 stories.

## Challenges and Growth Areas

### 1. Docker unavailable during development (Critical)
Stories 4.2 and 4.3 were authored without a running Docker stack. The Alembic migration chain was written manually and never validated against a live Postgres instance. This produced a `0003 missing revision` crash on first real startup — a bug that only surfaced at demo time, not during story sign-off.

**Impact:** Required elephant to debug and fix before the stack could start. Any Epic 5 story that assumes Postgres-backed repos would have hit this immediately.

### 2. Zero web mode testing during the epic (Critical)
All stories were developed and validated in native mobile mode only. Running the app in Expo web mode during the retrospective uncovered 7 issues that had been invisible throughout the epic:

| Issue | Root Cause | Fixed In Retro |
|---|---|---|
| Login "connection error" | Wrong password (seed: `correct-horse-battery-staple`) | ✅ Identified |
| Onboarding crash at completion | `Haptics.notificationAsync` not available on web | ✅ Fixed |
| Create class fails silently | `Haptics.impactAsync` in CameraFab throws before API call | ✅ Fixed |
| Class card navigation reverts instantly | `AuthGuard` calls `router.replace(homePath)` on every segment change | ✅ Fixed |
| Add student sheet stuck on onboarding class | Caused by CameraFab haptics crash corrupting React event queue | ✅ Fixed (CameraFab fix) |
| CSV import returns 422 | React Native `{ uri, name, type }` FormData pattern invalid in browser | ✅ Fixed |
| Tab press haptics crash (teacher/student/parent layouts) | `Haptics.selectionAsync()` not available on web | ✅ Fixed |

**Root cause:** No web smoke test step in the story definition of done.

### 3. AuthGuard redirect bug — invisible on native, critical on web
The `AuthGuard` in `app/_layout.tsx` ran on every `segments` change and called `router.replace(homePath)` for any authenticated user not in auth/onboarding — including users already inside their home group. On native, this is absorbed silently by the tab navigator. On web, it caused a genuine navigation redirect away from every detail screen.

This bug existed since Story 4.5 and would have broken all of Epic 5's deep navigation (grading screens, photo review, score adjustment) on web.

**Fix applied:** Added `inHomeGroup` check — only redirect if the user is not already within their role's root segment.

### 4. Spec-to-reality dependency drift (3 stories affected)
- Story 4.5: Design token names in spec (`surfaceDim`, `textMuted`) didn't match actual `@ilm/design-tokens` exports (`surfaceSecondary`, `textTertiary`)
- Story 4.6: Story assumed Google Sign-In SDK v14; actual package was v16 with breaking API changes
- Stories generally: `expo-haptics` assumed native-only usage but was called in all shared navigation paths

**Impact:** Invisible per-story friction. Devs discovered and compensated, but the spec remained inconsistent with reality.

### 5. In-memory repositories never replaced
All API tests pass and all story ACs are green, but the Postgres-backed repositories are not wired. Data resets on every API restart. This was an explicit deferral, but it creates a misleading picture of production-readiness.

**Epic 5 impact:** Any story that assumes persistent cross-session data will require the SQLAlchemy-backed repos to be connected first.

### 6. Role UX drift (parent and student roles thinner than teacher)
The teacher role received the most detailed story specs and the most complete implementation. Parent and student mobile screens are mostly placeholder stubs. While noted as intentional (teacher-first), this created noticeable inconsistency across roles in the running app.

## Significant Discoveries

### AuthGuard Navigation Bug
The `AuthGuard` in `app/_layout.tsx` contained a logic error where authenticated users were redirected to `homePath` on every `segments` change, not only when outside their home group. This was invisible on native (tab navigator absorbs the replace silently) but caused every programmatic navigation deeper than the root tab screen to instantly revert on web. Bug existed since Story 4.5 and affected all roles.

### Web/Native FormData Divergence
React Native's `{ uri, name, type }` FormData pattern for file uploads does not work in browser contexts. On web, `expo-document-picker` returns `blob:` URIs that must be fetched to get a real `Blob` before appending to `FormData`. This pattern will apply to any future file upload feature (e.g., Epic 5 assignment photo capture).

### In-Memory Repo Gap
The project has 304 passing API tests, all running against in-memory repositories with no Docker dependency. These tests provide strong unit-level confidence but zero integration confidence. The first end-to-end stack run (this retro session) caught issues the test suite could never have caught.
